🧠 The Enrich — How Cortex Gets Smarter
The Problem
A team member asks: "@neo what is StreamCore?"

Cortex searches 20GB of documents. Finds some raw chunks — a few sentences from a design doc, a table header from an Excel. Sends it to GPT-4.1.

GPT gives an answer, but it's thin and fragmented — bits and pieces stitched together. Not a real explanation. The team member walks away unimpressed.
Every question is logged
Cortex logs every question in the question_log table — who asked, when, what the answer was, and how confident the system was.
Trinity steps in — "ENRICH ON"
Neo says "ENRICH ON". Trinity (Opus 4.6) wakes up and does this:

1. Reads question_log — finds questions that got weak answers
2. Identifies the weak topics — "StreamCore got asked 4 times but the answer was always thin"
3. Goes to the actual source documents — reads the full HLD, the design doc, the architecture PowerPoint, not just tiny chunks
4. Writes a deep, flowing explanation — like a senior engineer would explain it to a new joiner
5. Saves it back into Cortex's database as an enriched document
Next time the same question is asked
Now when someone asks "what is StreamCore?" — Cortex finds the enriched doc first (it's scored higher than raw chunks).

GPT-4.1 gets a complete, well-written explanation instead of fragmented pieces. The answer is now confident, detailed, and accurate.

The team member says: "wow, this actually knows our stuff."
The learning loop
Weak answer → logged → Trinity reads source docs deeply → writes enrichment → strong answer next time

Every ENRICH run makes Cortex smarter. It's not retraining the AI — it's giving the AI better source material to work with.

20 enrichments written so far across 3 accounts. Each one transforms a "meh" answer into a "that's exactly what I needed" answer.
