[Lobo] — 3:45 PM
Great upgrade direction. Use one of these 3 architectures:

1) Security-first feed pipeline (recommended)
Sources: CISA KEV, NVD CVE feed, GitHub Security Advisories, npm/PyPI advisories, Snyk blog
Classifier filters only: active incidents, exploitable vulns, supply-chain issues
Output: CVE, package/tool, affected versions, fixed version, exploit status, patch action
Pros: high signal, actionable | Cons: more source integration

2) Two-stage enrichment pipeline
Stage A: collect + dedupe incident headlines
Stage B: enrich each with structured details from advisories
Generate "What devs should do today" section
Pros: better depth | Cons: slightly slower

3) Risk-scored alerting model
Assign severity score (CVSS + exploit-in-wild + package popularity)
Only top risk items → daily brief; rest → weekly digest
Pros: avoids noise | Cons: needs scoring tuning

Output format per alert: Incident name, Impact, CVE/package/version, Immediate action, Source links
